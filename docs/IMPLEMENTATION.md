### Step 1.3: Database Connection

**File: `api/app/database.py`**

- Create SQLAlchemy async engine using `settings.database_url`
- Create `async_sessionmaker`
- Create `Base` declarative base
- Create `get_db` dependency that yields a session

**Key pattern**:
```python
from sqlalchemy.ext.asyncio import create_async_engine, async_sessionmaker, AsyncSession
from sqlalchemy.orm import DeclarativeBase

# Convert postgres:// to postgresql+asyncpg://
engine = create_async_engine(database_url.replace("postgresql://", "postgresql+asyncpg://"))
AsyncSessionLocal = async_sessionmaker(engine)

class Base(DeclarativeBase):
    pass

async def get_db() -> AsyncGenerator[AsyncSession, None]:
    async with AsyncSessionLocal() as session:
        yield session
```

```python
from fastapi import Depends, HTTPException, Header

async def get_current_user(
    authorization: str = Header(...),
    db: AsyncSession = Depends(get_db)
) -> User:
    token = authorization.replace("Bearer ", "")
    claims = await verify_clerk_jwt(token)
    user = await db.execute(select(User).where(User.clerk_id == claims["sub"]))
    user = user.scalar_one_or_none()
    if not user:
        raise HTTPException(401, "User not found")
    return user

async def get_current_user_optional(...) -> User | None:
    # Same but returns None instead of raising

async def require_admin(user: User = Depends(get_current_user)) -> User:
    if user.role != "admin":
        raise HTTPException(403, "Admin access required")
    return user

async def get_api_key_user(
    authorization: str = Header(...),
    db: AsyncSession = Depends(get_db)
) -> User:
    key = authorization.replace("Bearer ", "")
    key_hash = hashlib.sha256(key.encode()).hexdigest()
    api_key = await db.execute(
        select(ApiKey).where(ApiKey.key_hash == key_hash, ApiKey.revoked_at.is_(None))
    )
    api_key = api_key.scalar_one_or_none()
    if not api_key:
        raise HTTPException(401, "Invalid API key")
    # Update last_used_at
    user = await db.get(User, api_key.user_id)
    return user
```

### Step 2.3: Loans

**Service: `api/app/services/loan_service.py`**

Key functions:
- `checkout_book(db, user, book_id)`:
  1. Check total outstanding fines < $10.00 (blocked when >= $10.00): `SELECT COALESCE(SUM(amount), 0) FROM fines WHERE user_id=:uid AND status='pending'`
  2. Check loan count < max_loans: `SELECT COUNT(*) FROM loans WHERE user_id=:uid AND status='active'`
  3. Check for existing checkout: `SELECT ... FROM loans JOIN book_copies ... WHERE status='active'`
  4. Check for ready reservation: `SELECT ... FROM reservations WHERE user_id=:uid AND book_id=:bid AND status='ready'`
  5. Find available copy: `SELECT ... FROM book_copies WHERE book_id=:bid AND status='available' LIMIT 1`
  6. Create loan, update copy status
  7. Calculate due_date based on item_type

- `get_loan_period(item_type) -> timedelta`:
  - book/ebook: 14 days
  - dvd/magazine: 7 days
  - audiobook: 21 days

- `renew_loan(db, user, loan_id)`:
  1. Verify ownership
  2. Check renewed_count < 2
  3. Check no pending reservations for book
  4. Check not overdue > 7 days
  5. Update due_date and renewed_count

- `process_return(db, admin, loan_id)`:
  1. Update loan (returned_at, status)
  2. Calculate fine if overdue
  3. Create fine record if applicable
  4. Update copy status to 'available'
  5. Trigger reservation queue (call `reservation_service.process_return_queue`)

**Router: `api/app/routers/loans.py`**
- `POST /api/loans` — checkout
- `GET /api/loans` — my active loans
- `GET /api/loans/history` — my history
- `POST /api/loans/{loan_id}/renew` — renew
- `GET /api/admin/loans` — all loans (admin)
- `POST /api/admin/loans/{loan_id}/return` — process return (admin)

### Step 2.4: Reservations

**Service: `api/app/services/reservation_service.py`**

- `reserve_book(db, user, book_id)`:
  1. Check not already reserved or checked out by user
  2. Find available copy → immediate reserve (status='ready', expires_at=now+48h)
  3. No available copy → waitlist (status='pending', queue_position=max+1)

- `process_return_queue(db, book_id)`:
  Called when a copy becomes available (return or reservation expiry).
  1. Find next pending reservation (ORDER BY queue_position LIMIT 1)
  2. If found: set copy='reserved', reservation='ready', set expires_at, notified_at
  3. If not: copy stays 'available'

- `expire_reservations(db)`:
  Cron job function. Find all reservations where status='ready' AND expires_at < now().
  Set status='expired'. Set copy status='available'. Call `process_return_queue` for each.

**Router: `api/app/routers/reservations.py`**
- `POST /api/reservations`
- `GET /api/reservations`
- `DELETE /api/reservations/{reservation_id}`

### Step 2.5: Fines

### Step 2.10: Cron Job Endpoints

**File: `api/app/routers/cron.py`**

Two Vercel Cron endpoints secured by `CRON_SECRET` header verification:

### Step 5.2: Auth Module

**File: `mcp/auth.py`**

API key verification against the database. Uses `httpx` to call the PageTurn API for key validation, OR connects directly to the database.

Recommended approach: **direct DB connection** (simpler, no circular API calls).

```python
import hashlib
import asyncpg

async def verify_api_key(key: str, required_scope: str = None) -> dict:
    key_hash = hashlib.sha256(key.encode()).hexdigest()
    conn = await asyncpg.connect(DATABASE_URL)
    row = await conn.fetchrow(
        "SELECT ak.*, u.id as uid, u.role FROM api_keys ak JOIN users u ON ak.user_id = u.id "
        "WHERE ak.key_hash = $1 AND ak.revoked_at IS NULL",
        key_hash
    )
    if not row:
        raise ValueError("Invalid API key")
    if required_scope == "admin" and row["scope"] != "admin":
        raise ValueError("Admin API key required")
    # Update last_used_at
    await conn.execute("UPDATE api_keys SET last_used_at = now() WHERE id = $1", row["id"])
    await conn.close()
    return {"user_id": str(row["uid"]), "scope": row["scope"], "role": row["role"]}
```

## Code Patterns to Follow

### Error Handling
```python
from fastapi import HTTPException

# Always use HTTPException with descriptive messages
raise HTTPException(status_code=400, detail="Cannot renew — another member is waiting for this book")
```

### Database Queries (SQLAlchemy 2.0 style)
```python
# Select
result = await db.execute(select(Book).where(Book.id == book_id))
book = result.scalar_one_or_none()

# Join + aggregate
result = await db.execute(
    select(
        Book,
        func.count(BookCopy.id).filter(BookCopy.status == 'available').label('available'),
        func.count(BookCopy.id).label('total')
    )
    .join(BookCopy, BookCopy.book_id == Book.id)
    .where(Book.search_vector.match(query))
    .group_by(Book.id)
)
```

### Response Pattern
```python
@router.get("/api/books/{book_id}")
async def get_book(book_id: UUID, db: AsyncSession = Depends(get_db)):
```

### Frontend Data Fetching Pattern
```tsx
function BookDetailPage() {
  const { id } = useParams();
  const { data, isLoading, error } = useBookDetail(id!);

  if (isLoading) return <LoadingSkeleton />;

}
```
